# TrustedHTML في جافا سكريبت: الأمان في معالجة المحتوى الديناميكي

## ملخص
TrustedHTML هو واجهة برمجية في JavaScript تهدف إلى تعزيز أمان معالجة المحتوى الديناميكي، حيث تسمح بتحديد المحتوى الموثوق به ورفض المحتوى الضار الذي قد يؤدي إلى هجمات مثل XSS (Cross-Site Scripting).

## الوثائق
### الغرض
تم تصميم TrustedHTML لتوفير وسيلة آمنة للتعامل مع محتوى HTML الديناميكي. تواجه تطبيقات الويب تحديات كبيرة في أمان المحتوى، وTrustedHTML يساعد المطورين على حماية تطبيقاتهم من تهديدات الأمان.

### الاستخدام
لإنشاء كائن TrustedHTML، يتم استخدام واجهة `TrustedHTML` المتاحة ضمن سياقات معينة، مثل `TrustedTypes`. يجب على المطورين تحديد سياسات موثوقة لتجنب المحتوى غير الموثوق به.

### التفاصيل
1. **إنشاء سياقات موثوقة**: يجب على المطورين إنشاء سياسات موثوقة باستخدام `TrustedTypes.createPolicy`.
2. **تحديد المحتوى الموثوق**: بعد إنشاء السياسة، يمكن للمطورين استخدام طرق مثل `policy.createHTML` لإنشاء كائن TrustedHTML.
3. **تطبيق المحتوى الموثوق**: يمكن استخدام كائنات TrustedHTML بأمان في إدراج المحتوى في DOM.

## الأمثلة
### مثال 1: إنشاء سياسة TrustedTypes
```javascript
if (window.TrustedTypes) {
    const policy = TrustedTypes.createPolicy('myPolicy', {
        createHTML: (input) => {
            return input; // تأكد من أن الإدخال موثوق
        }
    });
}
```

### مثال 2: استخدام TrustedHTML
```javascript
const safeHTML = policy.createHTML('<div>محتوى موثوق</div>');
document.body.innerHTML = safeHTML; // إدراج المحتوى الموثوق
```

## الشرح
### pitfalls
- **عدم التحقق من المحتوى**: يجب التأكد من أن المحتوى المدخل في `createHTML` موثوق به.
- **الاعتماد على المتصفحات**: ليس كل المتصفحات تدعم Trusted Types، لذا يجب اختبار الوظائف في مختلف البيئات.
- **التعامل مع مكتبات خارجية**: عند استخدام مكتبات خارجية، تأكد من أنها متوافقة مع Trusted Types.

### ملاحظات إضافية
- توفر TrustedHTML طريقة فعالة للحماية ضد هجمات XSS، ولكن يتعين على المطورين أن يكونوا على دراية بكيفية استخدامها بشكل صحيح.
- مع تزايد استخدام JavaScript في تطبيقات الويب، يصبح استخدام TrustedHTML ضرورة لضمان أمان التطبيقات.

## ملخص بجملة واحدة
TrustedHTML هو إطار عمل في JavaScript يعزز أمان معالجة المحتوى الديناميكي من خلال توفير آلية للتعامل مع المحتوى الموثوق به، مما يقلل من مخاطر هجمات XSS.
