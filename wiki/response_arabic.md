# استجابة (Response) في جافا سكريبت: مفهوم وأمثلة

## ملخص
استجابة (Response) في جافا سكريبت تشير إلى الكائن الذي يمثل استجابة الشبكة من طلب HTTP. تُستخدم استجابة لجلب البيانات من الخادم ومعالجتها في تطبيقات الويب.

## الوثائق
### الغرض
كائن الاستجابة هو جزء أساسي من واجهة برمجة التطبيقات (API) الخاصة بـ Fetch، التي تُستخدم لإجراء طلبات HTTP. يُتيح لك كائن الاستجابة الوصول إلى محتوى الاستجابة، مثل النصوص، بيانات JSON، والبيانات الثنائية.

### الاستخدام
عند إجراء طلب باستخدام دالة `fetch()`، يتم إرجاع كائن استجابة. يمكن استخدام هذا الكائن لمعالجة البيانات المستلمة من الخادم.

### التفاصيل
يتضمن كائن الاستجابة عدة خصائص وطرق، مثل:
- **status**: كود الحالة HTTP (مثل 200، 404، 500).
- **statusText**: نص كود الحالة.
- **headers**: كائن يحتوي على رؤوس الاستجابة.
- **body**: يمثل محتوى الاستجابة.
- **json()**: طريقة لتحويل محتوى الاستجابة إلى كائن JSON.
- **text()**: طريقة لتحويل محتوى الاستجابة إلى نص.

## الأمثلة
### مثال 1: استخدام `fetch` للحصول على بيانات JSON
```javascript
fetch('https://api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error('Network response was not ok');
    }
    return response.json();
  })
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error('There was a problem with your fetch operation:', error);
  });
```

### مثال 2: استخدام `fetch` للحصول على نص
```javascript
fetch('https://example.com/textfile.txt')
  .then(response => response.text())
  .then(text => {
    console.log(text);
  })
  .catch(error => {
    console.error('Error fetching the text file:', error);
  });
```

## الشرح
### الفخاخ الشائعة
- **فحص حالة الاستجابة**: يجب دائمًا فحص خاصية `response.ok` للتأكد من أن الطلب قد تم بنجاح قبل محاولة معالجة البيانات.
- **عدم معالجة البيانات بشكل صحيح**: تأكد من استخدام الطرق المناسبة (`json()`, `text()`, إلخ) حسب نوع المحتوى المتوقع.
- **التعامل مع الأخطاء**: استخدم `catch` للقبض على الأخطاء التي قد تحدث أثناء عملية fetch.

## ملخص من سطر واحد
استجابة (Response) في جافا سكريبت تمثل الكائن الناتج عن طلبات HTTP وتُستخدم لمعالجة البيانات المستلمة من الخادم.
